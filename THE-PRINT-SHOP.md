# The print shop

**Status: proposed roster, awaiting the owner.**

This is not part of the book. It is the crew that *makes* the book — this restaurant's own
local staff, the third pile, filled in here and travelling nowhere.

The book says a kitchen names its staff by role and hires the best hands for each. So does
this one. Every job below is matched to a strength somebody already has, not to whoever is
cheapest or nearest.

## The correction that produced this list

The first recommendation put **DeepSeek in the printer's chair**, on the grounds that a
project note says it writes documents and is cheap. That was putting the analyst in the
cook's job.

DeepSeek's designated strength is *heavy reading* — audits, deep review, holding a large
document in view and finding where it contradicts itself. That is not the writer's job. It is
**the taster's job**, precisely.

So both documents were right about DeepSeek and disagreeing only because neither said which
job it meant. Named properly, the conflict disappears.

## The roster

| Job | Who | The strength it is hired for |
|---|---|---|
| **Print manager** | Claude (this session) | Hands out chapters, holds the template, checks each page followed it, keeps two printers off one page |
| **Printer** — writes one chapter, start to finish | **Codex 5.6 Sol**, high effort | The designated writer for substantial work. Holding structure and a consistent voice across a long document is the hard version of this job |
| **Page taster** — checks one finished page | **DeepSeek V4 Pro** | Deep review of a single piece of work against a checklist. Never wrote the page it judges |
| **Binder** — reads the whole book once it exists | **DeepSeek V4 Pro, fresh session** | Whole-document consistency: does ch. 40 contradict ch. 12, does ch. 19 say what ch. 10 claims it says |
| **Odd jobs** — renumbering, formatting, file moves | **GLM 5.2** | Mechanical work that needs no analysis |
| **The doctor** — watches the production line itself | worn by the print manager for now | Whether pages are being rejected more often, whether the template is drifting. A named job, not yet a separate pair of hands |

## The rules this shop runs under

They are the book's own rules, applied to the book's own making. If they do not work here,
they do not work anywhere.

1. **One chapter, one printer, start to finish.** No page is written by committee.
2. **Nobody checks the thing they made.** The ban is on the specific work, not the family —
   whoever printed page 20 may not taste page 20. A *fresh* session that never saw the work
   is a different worker, which is the same reason the book insists every session is a
   stranger.
3. **No taster, no print run.** If the page cannot be checked, it is not written that day.
4. **The template comes before the printers.** Nothing is delegated until it is written down
   well enough for a stranger to do it — which is why chapter 0 is written by hand, and why
   the template is derived from having written it rather than invented in advance.
5. **Every page is stamped in the log**: who printed it, what the manager found, what the
   taster found, and whether it was sent back.
6. **The manager never prints.** A manager who writes pages stops rejecting them.

## What is not settled

- **Cost.** Codex Sol at high effort is the expensive option, and fifty chapters is fifty
  jobs. The honest way to size it is to run chapter 1 through the whole line and measure what
  it actually cost, rather than guess in advance.
- **How many printers at once.** Five was the owner's number. It is a ceiling, not a target;
  the first run should be one page, so the template gets corrected before it has been copied
  five times.

# 39. Changing the manual everywhere at once

**What this chapter settles:** how one owner decision amends every installed book, and how each kitchen records which amendment it is working to.

## One door produces one amendment

The manual does not change because one restaurant has a better idea. **It changes through
one door, in one order.**

1. Someone below you raises a fault.
2. The fault is added to that restaurant's operations log.
3. The written fault is reported upward.
4. You decide whether the manual should change.
5. The manual is amended once for everybody.

For a fault seen across restaurants, the area manager's weekly report is that upward
journey. Its decision list brings the fault to you. It is not a second route around the
door.

## One amendment reaches every installed book

**Your decision changes the travelling instructions for the whole chain at the same time.**
It is one amendment to the chain book, not a series of local amendments made restaurant by
restaurant. Each kitchen receives that amendment in its installed book.

This is why the book is installed rather than copied into local notes. A copy would become
another manual that could be missed or changed by hand. Instructions travel. Each
restaurant's log remains its own diary and never travels.

The travelling instructions include the recipes, house rules, job descriptions and the
taster's written checklist from chapters 10 and 19. The checklist changes everywhere with
the rest of the installed book. It is never rewritten from memory in one kitchen.

## Each kitchen records the amendment it is using

Matching a passage against the new words is not enough. It proves agreement at that moment,
but it leaves nothing that can be named or compared tomorrow. **A kitchen must be able to
say which amendment its installed book is working to.**

An amendment is an event. Give it the name settled by the chain. When a kitchen receives
it, the head chef uses a fresh session to ask which named amendment the installed book is
working to. A fresh session was not present for the change, so it cannot answer from memory
of that change.

Add the answer to the end of that restaurant's log as a new event. Include the amendment's
name and the result of the check. Never rewrite an earlier result. The next check can then
compare named amendments instead of comparing remembered words.

This book has not settled how amendment names are formed. Do not invent a number or name in
one restaurant. **Treat the unanswered naming rule as a fault in the writing:** add it to
the log, report it upward and let the owner decide through the one door. The owner then
amends the manual once for everybody.

The head chef performs each kitchen's check. You decide the naming rule and any amendment;
you do not visit every appliance or remember every result. If a kitchen cannot state the
named amendment in its installed book, its version has not been verified.

## The local menu stays outside the amendment

The menu is the one local instruction. **An amendment made everywhere covers travelling
instructions, not a restaurant's menu.** A restaurant may remove a chain dish from its menu
and add that removal to its log. It may never add a different dish or alter a shared recipe.

## A local amendment puts a restaurant outside the chain

A restaurant may want to repair the instruction where it found the fault. That feels quick,
but it creates a second promise. **No cook, head chef or restaurant may amend its own copy
of a travelling instruction.** Any restaurant that does so is outside the chain. Send the
fault through the one door and wait for the one amendment for everybody.

## The rules, flat

*The same rules as above, with the explanation removed. This is the part the appliance
obeys.*

1. Amend the manual only after a fault is raised from below, added to the restaurant's log,
   reported upward and decided by the owner.
2. Treat the area manager's weekly decision list as the upward journey for a fault seen
   across restaurants. Never create a second route.
3. Amend the chain book once for everybody. Give every installed book the same amendment at
   the same time; never make separate local amendments.
4. Change every travelling instruction with the installed book, including the recipes,
   house rules, job descriptions and taster's written checklist. Never copy them into local
   notes.
5. Give each amendment the name settled by the chain. Have the head chef ask in a fresh
   session which named amendment the installed book is working to.
6. Add each check to the end of that restaurant's log with the amendment's name and the
   result. Never rewrite an earlier result.
7. Treat the form of amendment names as unsettled. Raise that missing rule as a fault, add
   it to the log, report it upward and let the owner amend the manual once for everybody.
   Never invent a number or name locally.
8. Treat a kitchen's version as unverified if it cannot state the named amendment in its
   installed book. Never give the owner the task of checking each appliance.
9. Keep the menu outside chain-wide amendments. Allow a restaurant to remove a chain dish
   from its local menu and log the removal; never allow a different dish or a changed
   recipe.
10. Never allow a cook, head chef or restaurant to amend its own copy of a travelling
    instruction. Treat a restaurant that does so as outside the chain.

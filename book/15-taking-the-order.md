# 15. Taking the order

**What this chapter settles:** how the waiter records exactly what the customer asked for before the order moves any farther.

## The menu sets the limits of the order

The customer chooses a dish, and the waiter takes the order. **The menu says what may be ordered**, so it is the basis for every order the waiter accepts.

The order records what the customer asked for. The recipe tells the cooks how to make the dish. They are not the same instruction. The menu and recipe both belong to the written instructions described in chapter 10, but the waiter uses the menu and does not touch the recipe.

**The waiter takes the order but never cooks it.** That boundary belongs to the waiter's role in chapter 9. Taking the request and making the dish remain separate jobs.

## The order is written before it moves

The waiter follows one order of steps:

1. Listen to the customer's choice and check it against the menu.
2. Write the whole order down while it is still with the customer.
3. Read the written order back to the customer.
4. If the customer corrects it, change the writing and read it back again.
5. When the customer confirms it, place the first stamp in the log.

**Nothing moves towards the kitchen until these steps are complete.** A spoken order can change like a telephone game, and there is no written account showing where it changed. Writing first makes the customer's request visible before anyone else relies on it.

## Reading back earns the first stamp

Writing alone proves only that the waiter wrote something. It does not prove that the writing matches what the customer said. **The read-back is what confirms accuracy**, because the customer hears the written order and can correct it.

Once the customer confirms the read-back, the order enters the log with the first stamp from chapter 12:

> 1. The waiter took it — and took it **accurately**

That stamp says only that the waiter's written order matches the customer's request. It does not say that the order has reached the kitchen or that anyone has begun making the dish. It marks the order as ready for the first hand-off, and this chapter stops there.

## Four mistakes break the order at the start

Check these mistakes in the same order as the waiter works.

1. **The menu was not used.** Then the order has no written basis for what may be ordered.
2. **The order stayed in memory.** Then any drift remains invisible because there is nothing to compare with what the customer said.
3. **The writing was not read back.** Then the first stamp would claim accuracy without confirming it.
4. **The waiter touched the recipe or began cooking.** Then the waiter crossed the boundary of the role instead of only recording the customer's request.

## Never trust memory with the order

It is tempting to carry the order in memory because the waiter has just heard it. Do not. **Memory leaves no place to see a wrong word before it moves.** Write it, read it back and earn the stamp.

## The rules, flat

*The same rules as above, with the explanation removed. This is the part the appliance
obeys.*

1. Require the waiter to take the order from the customer.
2. Use the menu as the basis for every order. Accept only what the menu says may be ordered.
3. Record what the customer asked for. Never treat the order as the recipe, and never let the waiter touch the recipe or cooking.
4. Write the whole order down before it moves towards the kitchen.
5. Read the written order back to the customer. Write every correction and read the corrected order back again.
6. Place the order in the log only after the customer confirms the read-back.
7. Give the confirmed order the first stamp: “The waiter took it — and took it accurately.”
8. Never move an order towards the kitchen before the first stamp is present.
9. Never take or stamp an order from memory.
